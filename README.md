# Спринт 1 - Задание 1

## Уровень 1. Проектирование
Для проектирования был выбран феймворк Webpack Module Federation. Основной причиной выбора является то, что на работе упоминали его, как вариант для разбивки на микрофронтенды.

Основные преимущества Webpack Module Federation:
- Общие зависимости через shared, что позволит переиспользовать компоненты сразу в нескольких местах
- Lazy loading - загрузка только необходимых компонентов, что ускоряет работу и экономит трафик
- Модули можно развернуть независимо друг от друга
- Хорошо подходит для структуры этого проекта

Недостатки:
- Необходим Webpack для работы
- Проблемы с версиями общих библиотек

## Уровень 2. Планирование изменений
### Домены
В приложении сразу можно выделить три домена, которые решают свои задачи:
- Авторизация
- - Регистрация нового пользователя
- - Вход и выход пользователя из приложения
- - Проверка статуса авторизации пользователя
- - Управление токеном авторизации
- Профиль пользователя
- - Вывод информации о пользователе
- - Изменение имени пользователя
- - Изменение аватара пользователя
- Каталог мест
- - Вывод списка мест
- - Добавление и удаление места
- - Просмотр фотографии места

### Микрофронты

Микрофронты повторяют структуру папок оригинального фронта, изменения в коде минимальны, но достаточны, чтобы код можно было запустить

## Уровень 3. Запуск готового кода

Проверить работоспособность кода, можно запустив все микрофронтенды в dev-режиме:
- auth
- profile
- cards
- mesto

#### Auth
```
cd frontend\microfrontend\auth
npm install
npm start
```
#### Profile
```
cd frontend\microfrontend\profile
npm install
npm start
```
#### cards
```
cd frontend\microfrontend\cards
npm install
npm start
```
#### mesto
```
cd frontend\microfrontend\mesto
npm install
npm start
```

Сайт будет доступен по адресу ```http://localhost:8088/```
