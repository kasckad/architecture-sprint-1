## Этап проектирования
### Выделенные для разделения на микрофронтенды доменные области   
- Аутентификация и авторизация
  - Логин
  - Регистрация
- Галерея
  - Показ фото
  - Учет лайков
  - Удаление фото
- Профиль пользователя
  - Редактирование профиля
  - Загрузка новых фото

### Метод реализации
- Runtime с клиентской композицией   
Метод интеграции микрофронтендов выбан в связи с тем, что профиль пользователя связан с галереей и нужно обеспечить интерактивное взаимоействие. Большая часть текущего функционала связана с интерфейсной частью приложения

### Фреймворк
- Webpack Module Federation

Выбран в связи с большим коммьюнити и возможностью использовать общие модули, например `api.js`

#### Микрофронтенды    
- auth
- gallery (корневой)
- profile

### Связь между компонетами
- Глобальное состояние.
- Библиотека Redux

Подход с глобальным состоянием выбран из-за связанности компонентов и необходимости обновлять галерею при действиях в профиле и обеспечении большей гибкости.
При добавлении фотографий в профиле будет отправляться событие об этом в компонент галереи
