# Задание 1

## Проектирование

Для создания микрофронтендов был выбран Webpack Module Federation, так как: 
* монолит был написан с использованием одной технологии
* имеется необходимость использования общего кода во время выполнения
* осуществляется динамическая загрузка только нужных модулей

## Планирование изменений

Было принято решение разбить монолит на следующие микрофронтенды:
* host - содержит общий код для всех микрофронтендов
* auth - микрофронтенд, отвечающий за регистрацию и авторизацию пользователя
* card - микрофронтенд, отвечающий за управление карточками (создание, удаление, лайки)
* profile - микрофронтенд, отвечающий за управление профилем пользователя

В случае успешного развития сервиса и увеличения количества пользователей, можно рассмотреть вариант выноса управления лайками в отдельный микрофронтенд.
