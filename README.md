## Сервис Mesto

### Описание проекта
Данный проект позволяет пользователям делиться фотографиями, оценивать их, редактировать профиль. 
Также имеет систему аутентификации и регистрации пользователей.

### Разбивка проекта на микрофронтенды
При разбивке сервиса на микрофронтенды использовались стратегии вертикальной нарезки и изоляции.
С помощью DDD были выделены следующие бизнес функции:
- Аутентификация и регистрация
- Профиль пользователя
- Каталог фотографий

На основании этого было решено разделить проект на три микрофронтенда соответствующие этим бизнес функциям.
При этом в изначальном коде добавление фотографий в каталог находилось в компоненте профиля. Было принято решение
отнести эту функцию к микрофронтенду каталога фотографий.

Таким образом получившиеся микрофронтенды будут представлять собой самодостаточные системы. Также у них не будет 
общих зависимостей во время выполнения. 
Все компоненты/стили/скрипты были разнесены по микрофронтендам.

### Интеграция микрофронтендов
В данном проекте подошли бы и Single SPA и Module Federation. Было принято решение использовать Module Federation,
благодаря его более простой настройке и удобной работе с общими зависимостями.
Таким образом получилось 4 модуля: хост и три микрофронтенда (remote модули).
Контекст пользователя из основного приложения передается в модули как props.

### Запуск проекта
Задание выполнено на уровне 2. Проект не настроен для полноценного запуска.
