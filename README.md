
Монолит разнес на доменные области: 

- аутентиффицаия 
- управление профилем 
- карточки мест 

В рамках разделения использовал Module Federation. 

Получил 3 микрофронтенда: 

- аутентификации (Микрофронтенд аутентификации, который отвечает за вход пользователя в систему, а также за регистрацию и авторизацию во всем приложении)
- профиль пользователя (Микрофронтенд профиля пользователя, который обеспечивает редактирование пользовательских данных)
- карточка мест (Микрофронтенд карточек мест, который отвечает за операции CRUD с карточками мест)
