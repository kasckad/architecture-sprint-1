Выбор фреймворка для создания микрофронтендов

Проанализировав структуру проекта я выделил следующие особенности:
1. Основная технология, использованная для написания фронтенда - React (17.0.2)
2. Используется редирект по URL для Log in и Sign up
3. Некоторые копоненты переиспользуются несеолько раз. Например, PopupWithForm используется во всех компонентах с поп-апами (AddPlacePopup, EditAvatarPopup и тд)
4. В App.js создаются общие переменные состояния, такие как currentUser и email
5. В приложении используется общий слой с утилитами. API и auth используются всеми компонентами

Вывод:
Для разбиения приложения на микрофронтенды в данном случае больше подходит фреймворк Module Federation. Он хорошо зарекомендовал себя в случаях когда:
1. В приложении используется единый технический фреймворк. В данном случае приложение Mesto использует только React.js
2. Переиспользуются компоненты и модули. Mesto использует общий api и auth слой. Некоторые компоненты ссылаются на другие (как, например с Pop-up)
Также Module Federation позволит оптимизировать загрузку библиотек и динамически подгружать требуемые микрофронтенды. Это сэкономит ресурсы и оптимизирует бандл. 
Из минусов, Module Federation не очень хорошо оптимизирован для маршрутизации и разделения приложения на полностью независимые микрофронтенды. Но, в Mesto нет сложной маршрутизации, а полной независимости микрофронтендов не требуется из-за изначальной связанности компонентов. Также, при необходимости, маршрутизацию можно реализовать при помощи SSR.
Исходя из вышеперечисленного наиболее подходящим фреймворком для приложения Mesto является Module Federation
