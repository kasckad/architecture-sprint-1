# **Выполнение задания по спринту**

## * Задание 1

### 1. Проектирование
Целью разбития проекта на мирофронтенд архитектуру является:
1. Независимая разработка и развертывание
2. Разделение зон ответственности команд
3. Масштабирование приложения
### 2. Планирование изменений
Были выявлены основные домены:
1. [ ] **auth** - модуль управления авторизацией пользователей
2. [ ] **profile** - модуль управления профилем пользователя
3. [ ] **cards** - модуль управления карточками
4. [ ] **host** - хост модуль управления приложением

Для реализации архитектуры микрофронтендов, был выбран framework ModuleFederation, 
так как нет необходимости использования разных стеков в микрофронтендов - все выпонлняется на React.

Компоненты разнес по микрофронтендам в соответствии с зоной ответственности каждого микрофронтенда.
Так же был выявлен общий элемент **PopupWithForm** который используется в большинстве компонентов.
Его предлагается вынести в общий пакет npm или как git-submodule.
Так как происходит тесное взаимодействие между компонентами, был внедрен
глобальный менеджер состояний Redux, что позволило использовать централизованное
управление состояниями.
### 3. Запуск готового кода
Проект полностью работоспособен и работает без ошибок.
npm install, npm run build и npm start ошибок в консоли не выдают
