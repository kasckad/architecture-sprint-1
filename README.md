## **1. Проектирование. Выбор фреймворка.**
Для реалиации микрофронтенд - архитектуры, мной был выбран инструмент **Webpack Module Federation**.
Выбор обусловлен относительной простотой использования и оптимизацией работы благодаря  функции lazy loading, а также тем, что
в даном проекте нам не требуется использование различных frontend-фреймвроков.

**Webpack Module Federation** позволит использовать различные версии библиотек в разных модулях, а использование общего
frontend-фреймврока облегчит общение и взаимодействие разработчиков из разных команд.


## **2. Планирование изменений. Структура кода**

Первоначальный монолитный фронтенд предполагается разбить на 4 самостоятельных микрофронтенд модуля.
Module Federation предполагает разбиение приложения на основной приложение(Хост) и удаленные модули.
Основной модуль - _host_, будет отвечать за динамическую загрузку удаленных модулей, а также за
основной лейаут и расположение элементов на нем.

### Список удаленных модулей:
**auth**
**Auth - модуль авторизации пользователя**, отвечает за авторизацию и регистрацию пользователя в приложении.  
_Основные компоненты модуля:_  
**Форма входа** - компонент _Login_, реализует авторизацию пользователя в приложении  
**Форма регистрации** - компонент _Register_, реализует регистрацию пользователя в приложении  
**Панель управления авторизацией** - компонент  _Header_, содержащий кнопки вход, выхода, и регистрации.  
**Информационный блок-подсказка** - компонент _InfoTooltip_, реализует функцию отображения системных подсказок при ошибках регистрации и авторизации.

**card**
**Card - модуль карточки места**, отвечает за добавление новых мест, отображение карточек и взаимодействие с ними  
_Основные компоненты модуля:_  
**Окно добавления новых мест** - компонент _AddPlacePopup_, модальное коно позволяющее добавить картинку и описание при создании нового места
**Карточка места** - компонент _Card_, форма карточки, позволяющая просматривать место, открывать модальное коно просмотра изображения,
лайкать и удалять карточку.
**Окно просмотра изображения** - компонент _ImagePopup_
**Окно-обертка формы** - компонент _PopupWithForm_, враппер для компонент, реализующих модальные окна с формами, в данном модуле - AddPlacePopup

**profile**
**Profile - модуль профиля пользователя**, отвечает за добавление редактирование профиля, смену аватара, а также отображение профиля на странице.  
_Основные компоненты модуля:_  
**Окно смены аватара** - компонент _EditAvatarPopup_, модальное коно позволяющее сменить аватар пользователя
**Окно редактирования профиля** - компонент _EditProfilePopup_, модальное коно позволяющее редактировать профиль пользователя
**Окно-обертка формы** - компонент _PopupWithForm_, враппер для компонент, реализующих модальные окна с формами, в данном модуле - EditAvatarPopup, EditProfilePopup
**Блок профиля** - компонент _ProfileMain_
