# Задание 1. Разделение фронтенда

## Этапы

### 1. Выделение функций

В приложении есть несколько основных функций:

- Вход и Регистрация
- Отображение и редактирование профиля
- Загрузка нового места
- Отображение каталога мест
- Отметка мест которые понравились

Для разделение приложения я планирую использовать стратегию вертикально нарезки.
Разделение будет на такие блоки:

- Работа с Местами
- Работа с профилем, а так же регистрация и вход
- Основное приложение для сборки и синхронизации общей памяти

Это обусловлено тем, что приложение содержит четкие и видимые функции которые удобно разделить

### 2. Метод реализации

Все микрофронтенды можно оставить в рамках одно фреймворка - React, так как неизвестно сколько человек будет над разработкой.
Использование нескольких фреймворков может замедлить разработку если команда состоит из небольшого количества человек,
а монолитное приложение уже сделано на React.

### 3. Выбор инструментов для создания

Для сборки микросервисов я использовал Webpack Module Federation
